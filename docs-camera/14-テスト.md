# テスト（Vitest）

ユニットテストの実行方法と、これから増やすときの計画メモ。
フレームワークは **Vitest**（`package.json` の `devDependencies` / `"test"` スクリプト）で導入済み。
追加インストールなしで動く。

関連: [doTest.sh](../doTest.sh) / [07-バージョン管理.md](07-バージョン管理.md)

## 使い方（`./doTest.sh`）

`guruguru-avatar/` で実行する。中身は Vitest の薄いラッパー（`npm test` と同じ実体）。

| コマンド | 動作 |
| --- | --- |
| `./doTest.sh` | 全テストを1回実行（CI 向け・`vitest run`） |
| `./doTest.sh watch` | 変更を監視して自動再実行（開発中・`vitest`） |
| `./doTest.sh <pattern>` | ファイル名フィルタで一部だけ（例 `./doTest.sh obs`） |
| `./doTest.sh -h` | ヘルプ |

`npm test`（= `vitest run`）でも同じだが、`watch` と絞り込みを1コマンドにまとめてある。

## いま動いているテスト

純関数を中心にカバー（2026-06 時点で 17 ファイル / 158 件）。例:

- `src/obs-mode.test.js` — `?obs` / `?shadow` の解析（三状態フラグ・rx 既定 ON）
- `src/relay-mode.test.js` — `?tx` / `?rx` / `?relay` の解析
- `src/face/*.test.js` — `avatar-state` / `apply-state` / `state-codec` / `relay-client` ほか

UI（React コンポーネント）やカメラ・描画は対象外。**URL 解析や状態変換のような副作用のない純関数**を優先的にテストしている。

## 仕組み

- テストファイルは実装の隣に置く（`src/**/*.test.js`）。
- 設定は `vite.config.js` を Vitest がそのまま読む（専用の `vitest.config.js` は無し）。
- 実行は Node 上（jsdom 等のブラウザ環境は今は使っていない＝純関数前提）。

## 段階的に増やす計画（今はやらない）

最初は「全実行・監視・絞り込み」だけで足りる。必要になったら次の順で足す。

1. **カバレッジ**

   ```bash
   npm i -D @vitest/coverage-v8
   ```

   その後 `./doTest.sh coverage` を `vitest run --coverage` として追加する
   （パッケージ未導入のまま `--coverage` を付けると対話プロンプトが出るので、導入とセットで実装する）。

2. **CI 連携**

   `pages.yml` のデプロイ前に `./doTest.sh` を挟み、グリーンでなければデプロイしない
   （[doDeploy.sh](../doDeploy.sh) の流れに組み込む）。

3. **E2E**

   描画・カメラ・OBS 受信（`?rx`）のような実ブラウザ挙動は Playwright で別レイヤにする。
   ユニットテストとは目的が違うので、必要が出てから最小構成で導入する。
