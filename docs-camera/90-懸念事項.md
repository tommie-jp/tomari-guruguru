# 懸念事項（既知・未対応メモ）

セキュリティ/プライバシー監査（2026-06-21）で確認した、**今回あえて修正していない**軽微な項目の備忘録。
いずれも LOW / INFO で機能破綻はないが、将来手を入れるときの参考に残す。

## 1. 軽微なリソースリーク（今回は未修正）

### talk-app: 音声 object URL を revoke していない

- 場所: `src/talk-app.jsx`（ファイル選択ハンドラ `onFilePick`）
- 症状: 音声ファイルを選ぶたびに `URL.createObjectURL(f)` を呼ぶが、対応する `URL.revokeObjectURL` がない。選び直すほど古い blob が解放されず蓄積する。
- 影響: 軽微（同一ページで何度もファイルを差し替えたときだけ増える）。
- 直すなら: 直前の URL を ref に保持し、差し替え時とアンマウント時に revoke する。

### talk-app: マイク stream / AudioContext をアンマウントで解放していない

- 場所: `src/talk-app.jsx`（`makeAudioEngine` まわり）
- 症状: `stopMic()` は `toggleMic` からしか呼ばれず、アンマウント時に stream / AudioContext を明示的に閉じる `useEffect` クリーンアップがない。
- 影響: 軽微（単一の長命ページなのでタブ/ナビゲーションで OS が回収する）。ただしマイク使用中インジケータが残ることがある。
- 直すなら: `useEffect(() => () => { engine.stopMic(); ctx?.close?.(); }, [engine])`。

### use-hand-pose: モデル読込中のアンマウントで recognizer がリーク

- 場所: `src/tracking/use-hand-pose.js`（`init()` の `Promise.all` 直後）
- 症状: モデル生成の `await` 解決前にアンマウントすると、`cancelled` 判定で `return` する経路があり、生成済みの recognizer を `close()` しない。
- 影響: 軽微（`tracking.html` で読込中（約1〜2秒）に離脱した場合や dev の StrictMode）。
- 直すなら: `await` 後に `if (cancelled) { gesture?.close?.(); pose?.close?.(); return; }`。

## 2. リレーのセキュリティ強化（公開する場合のみ）

中継サーバ（`server/relay.mjs`）は **ローカル/私設網での実行前提**として運用する方針
（[08-WS中継の接続手順.md](08-WS中継の接続手順.md) と `relay.mjs` 冒頭に明記済み）。
公開インターネットや不特定多数の LAN へ晒す必要が出たときは、以下を実装してから行う。

- トークン認証（接続 upgrade 時に `?token=` を定数時間比較して照合）
- ルーム / チャンネル ID（producer が自分の consumer にだけ届くよう分離）
- Origin allowlist（`WebSocketServer` の `verifyClient`）
- `maxPayload` 制限（状態フレームは数 KB で十分。例: `maxPayload: 16 * 1024`）と接続数・レート制限（未設定だと ws 既定 100 MiB で DoS 余地）

## 3. その他 LOW（許容中）

- **Google Fonts を SRI なしで CDN 読込**（`index.html` ほか）。実行時の唯一のサードパーティ資源。CSS のみで JS 実行はできないため影響は限定的。フォントを self-host すれば外部依存をなくせる。
- **postMessage リスナに origin チェックがない**（`src/tweaks-panel.jsx`）。受け取る `__activate_edit_mode` 等は boolean を切り替えるだけで、DOM / eval / 通信の sink に未信頼データは届かない。防御的に origin allowlist を足す余地はある。
