# 使い方

トマリぐるぐる（アバター）を **Webカメラで顔の向きを推定して同調させる** ための使い方メモ。

このフォークの主目的はカメラ版2（`index.html`・Pixi スプライト描画／複数アバター）。
マウス追従・マイク口パクのオリジナル機能もそのまま使える。

- 本体アプリは `guruguru-avatar/` サブディレクトリにある。**コマンドは原則そこで実行する**。
- パッケージマネージャは **npm**（`package-lock.json` 管理）。pnpm / yarn は使わない。
- Node.js は **20.19+ または 22.12+**（22 LTS 推奨）。

## セットアップ

```bash
cd guruguru-avatar
npm install
```

`npm install` 後、初回の `npm run dev` / `npm run build` 時に
`predev` / `prebuild` フックが MediaPipe アセットを `public/mediapipe/` に配置する。

- WASM 一式は `node_modules` からコピー（毎回上書き）。
- モデル（`face_landmarker.task` ほか）は Google ストレージから **初回のみダウンロード**。
  → 初回はネット接続が必要。`public/mediapipe/` は git 管理外なので都度再生成される。

手動で配置だけしたいときは:

```bash
npm run setup:mediapipe
```

## 起動

```bash
npm run dev
```

- 既定で `http://127.0.0.1:5173/` を起動し、`/`（index.html・カメラ版2）を自動で開く。
- Windows なら `start.bat` をダブルクリックでも起動できる。

### 各ページ

| URL | 内容 |
| --- | --- |
| `/` (index.html) | **カメラ版2（本命）** — Webカメラで顔の向きに同調・Pixi スプライト描画・複数アバター選択（サイトのトップ。旧 camera2.html） |
| `/talk.html` | トーク版 — マイク入力または音声ファイルで口パク |
| `/guruguru.html` | ぐるぐる版 — マウス追従で25方向に振り向く |
| `/tracking.html` | 手・ポーズ可視化 — 手指/全身骨格とジェスチャー名を表示するデバッグ用 |
| `/index_old.html` | `index.html` へリダイレクト（旧 index.html） |

### カメラ・マイクの前提

- Webカメラ（`getUserMedia`）とマイク入力は **`localhost` か HTTPS でのみ動く**。
  必ず `127.0.0.1` / `localhost` 経由で開く（IP直打ちは不可）。
- 初回アクセス時にブラウザのカメラ／マイク許可ダイアログが出るので **許可** する。
- Google Fonts を CDN から読むため、初回表示にはネット接続が必要。

### WSL2 + Windows Chrome で使う場合

`guruguru-avatar/.env.local`（git 管理外）で上書きする。`.env.local.example` をコピーして作成:

```bash
cp .env.local.example .env.local
```

| 変数 | 効果 |
| --- | --- |
| `VITE_HOST=1` | `0.0.0.0` でリッスンし、Windows 側 Chrome から `localhost:5173` で到達できる |
| `VITE_NO_OPEN=1` | 自動ブラウザオープンを無効化（WSL から Windows の Chrome を起動できないため） |

WSL の場合は **両方 1 がおすすめ**。Windows の Chrome で
`http://localhost:5173/index.html` を開く（`localhost` なのでカメラ・マイクも動く）。

## カメラ版の使い方（index.html）

カメラに顔を映すと、アバターが次の動きで同調する。

- **顔の向き** → 25方向（5×5）のフレームに変換して振り向く。
- **口の開き**（`jawOpen`）→ 口パク（とじ／はんびらき／ぜんかい の3段階）。
- **まばたき** → 実際の目の開閉に同調（既定 ON。OFF にすると自動まばたき）。
- **首かしげ**（roll）→ アバターを傾ける。
- **左右・上下スライド** → 顔の位置に合わせてアバターが画面内を移動。
- **距離ズーム** → カメラに近づくと拡大、離れると縮小。

画面右下の **調整パネル（Tweaks）** で各挙動を調整する。設定は localStorage に
保存され、次回も復元される。「設定をデフォルトに戻す」でリセット。

### 最初に合わせると良い3つの基準

自然な姿勢・目の大きさ・カメラ距離は人それぞれなので、現在の状態を基準として記録できる。

- **今の向きを正面にする** … 少し下や横を向いた自然な姿勢を「正面」に設定（正面バイアス）。
- **今の目の大きさを まばたきなし にする** … 細目やカメラ角度で
  開眼時でも「閉じ」判定が出る人向けの基準調整。
- **今の距離を基準にする** … 今のカメラ距離をズーム等倍（1.0）の基準にする。

各ボタンには対になる「リセット」がある。

### 主な調整項目

- **顔追従**: 感度 / 追従速度 / まばたき同調・感度
- **正面バイアス**: 左右・上下バイアス（°）
- **反転**: 左右反転 / 上下反転 / カメラ映像を表示（鏡像プレビュー）
- **首かしげ・スライド**: かしげ量・上限・支点・反転、左右/上下のスライド量・上限・反転、動きの滑らかさ
- **ズーム（カメラ距離）**: 距離でズーム ON/OFF、ズーム量・下限・上限
- **口パク**: 口の感度 / しきい値（はんびらき・ぜんかい）/ 口を閉じる速さ
- **見た目**: キャラサイズ / 背景色
- **デバッグ**: グリッド表示 / 表情係数（MediaPipe blendshapes）を表示

> 「カメラ映像を表示」を OFF にしても、顔検出のため映像要素自体は画面外で動き続ける。

## ビルドと確認

```bash
npm run build      # 本番ビルド (dist/ 出力)
npm run preview    # ビルド結果をローカル確認
npm run verify:pages   # GitHub Pages 向けビルドを検証
```

- ビルド時のみ base が `/guruguru-avatar/` になる（`vite.fork.js` で上書き）。
- `preview` は GitHub Pages と同じパスで起動する:

```text
http://127.0.0.1:4173/guruguru-avatar/index.html
http://127.0.0.1:4173/guruguru-avatar/talk.html
http://127.0.0.1:4173/guruguru-avatar/guruguru.html
```

## つまずいたら

- **カメラ／マイクが動かない**: `127.0.0.1` か `localhost`、または HTTPS で開いているか確認。
  IP 直打ち（`192.168.x.x` 等）では `getUserMedia` が使えない。
- **「顔が見つかりません」**: 明るさを確保し、顔全体がカメラに収まるようにする。
  検出中はステータスが緑（顔を検出中）になる。
- **初回の起動が遅い / モデルが落ちてこない**: ネット接続を確認。
  モデル DL は初回のみ。再生成は `npm run setup:mediapipe`。
- **向きがずれる・反応しすぎる**: 「今の向きを正面にする」で基準を取り、感度・追従速度を調整。
