# 2.5D レイヤーアバターの設計（自作 Live2D 風）

> **ステータス: 未実装・将来計画**
> ここに記載する `01-babylon` アバター・`RigModel`・各コンポーネントはまだ実装されていない。
> 現時点での設計メモ・PoC 計画であり、実装は段階的ビルド計画（PoC-1〜7）を経て順次行う予定。

可愛い2D絵に「表情（目を閉じる・口パク）」と「動き（髪揺れ・振り向き）」を、
表情ごとに何枚も絵を用意せずに付けるための設計メモ。Live2D / 3D ポリゴンを使わず、
Babylon.js 上に「2.5D レイヤー視差リグ」を自作する方針。

## なぜこの方針か（決定の経緯）

- **3D ポリゴンは不採用**: 現代技術でも可愛くしづらい。AI（ChatGPT-Image）で可愛い2D絵は作れる→ベースは必ず2D絵。
- **Live2D は不使用**: 個人・非収益でも SDK Release License の登録手続きが要り（MIT のような無手続きではない）、かつ Babylon 版ランタイムが事実上存在しない。
- **土台は Babylon**: 当初は「2D なら Pixi が軽い」と評価したが、2つの新事実で覆った。
  - tommieChat（`github.com/tommie-jp/tommie-chat`）が既に **Babylon.js 9 製の3D MMO**で、`AvatarSystem` が 5 レイヤー＋depth の重ねビルボードを実装済み。
  - 狙いが「2.5D レイヤー視差」で、**Babylon の3D空間（実 Z パララックス）が武器**になる。
  - → OBS と tommieChat の**両用途で実装を1つに統一**できる。

## 設計の核：2.5D レイヤー視差

1枚絵をパーツ分解し、各パーツに**実 Z（奥行き）**を与えて Babylon の3D空間に並べる。
`faceRoot`（親 TransformNode）を yaw/pitch/roll で微回転させ、**透視カメラの視差**で振り向きを出す。
角度ごとの絵は不要（中間角は計算で生成）。

- 透視（perspective）カメラ必須。ortho 不可・Sprite billboardMode 禁止（視差が消える）。
- パーツ Z 配置の例: 後ろ髪 z=-0.3 / 顔ベース 0 / 目・口 +0.02 / 前髪 +0.15。
- Live2D の3D回転設定（Center of Rotation / Z-Offset / perspective strength）と同原理。

## 表情を「枚数を増やさず」出す

| 表情 / 動き | 手法 | 追加画像 |
| ---- | ---- | ---- |
| 振り向き（yaw/pitch） | レイヤー Z 視差＋faceRoot 回転 | 0 |
| 首かしげ（roll） | faceRoot の Z 軸回転（既存 compensate-roll-for-yaw 流用） | 0 |
| まばたき（eyesClosed） | まぶた板の縦スケール、または開/半/閉 2-3 キーフォーム頂点ブレンド | 目 1 枚 |
| 口パク（mouth） | 閉/中/開 2-3 キーフォーム頂点を mouth で線形ブレンド | 口 2-3 形 |
| 髪揺れ | 前髪板（カメラ寄り Z）に verlet/バネ物理。yaw/pitch の加速度を外力に | 0 |
| 呼吸 / アイドル | 体ルートに低周波 sin の上下・微スケール | 0 |

既存の顔シグナル `yaw / pitch / roll / mouth / eyesClosed`（`src/face/derive-face-signals.js`）、
WS 中継、OBS 透過（`?obs=1`）はそのまま流用する。

## 自作するのは「再生」だけ。「制作」は OSS で省力化

完全無料でブラウザ成熟の再生ランタイムは存在しない（Inochi2D は Web ランタイム未完成、
Spine は有償ライセンス、pixi-live2d-display は Cubism Core 必須）。**再生は自作が妥当**。
ただし素材制作ツールは流用できる。

- **See-through**（Apache-2.0）: 1枚絵をパーツ別 RGBA に自動分解＋隠れ部分のインペイント。
- **StretchyStudio**（MIT）: 自動リグ・メッシュ・キーフォーム作成。
- ※ Spine ランタイム本体は使わず、書き出しデータだけ拝借する（ライセンス回避）。

## アーキテクチャ（描画エンジン非依存に切る）

| コンポーネント | 役割 | 技術 |
| ---- | ---- | ---- |
| LayerAssets（layers.json） | パーツ別 RGBA ＋描画順・Z 深度のメタ | See-through 出力＋手補正 |
| RigModel | param→変形の純データ層（エンジン非依存） | 自作 JS。param→キーフォーム重み線形補間。immutable 更新 |
| SceneHost | パーツ板を実 Z 配置・faceRoot 回転・透視カメラ | Babylon Plane/TransformNode/perspective カメラ/renderingGroupId |
| DeformLayer | 目=縦スケール、口=Morph Targets、髪=verlet | Babylon Morph Targets ＋自作 verlet |
| SignalBridge | 既存 face signal / WS 受信を RigModel へ結線 | use-face-pose / use-relay を無改変流用 |
| AvatarSelector | sheet 方式と rig 方式を併存 | character-config に `type:'sheet'｜'rig'` 追加 |

**肝**: RigModel をエンジン非依存に切り出しておく。万一 Pixi で軽く検証したくなっても土台を差し替え可能。

## OBS と tommieChat の二重デプロイ

両方 Babylon なので、リグを**共有 Babylon モジュール**にして両リポから import する
（tommieChat の `lib/babylon-rpgmaker-sprites/` と同じ流儀。例 `lib/babylon-face-rig/`）。

- **guruguru-avatar（OBS/CEF）**: `?obs=1` で全画面・透過・単体表示。`01-babylon` が苗床（透過・HW GPU 確認済み）。
- **tommieChat（MMO）**: 既存の `AvatarSystem` / `SpriteAvatarSystem` と並ぶ第3のアバター種別「rig」として追加。名前札・吹き出し・位置同期・AOI は既存配管に乗る。`AvatarSystem`（5 レイヤー＋depth）の進化形にもできる。
- **駆動 API**: `RigModel.setParams({yaw,pitch,roll,mouth,eyesClosed})` を公開。OBS=カメラ/WS 駆動、tommieChat=アプリから直接 or TTS/テキスト連動の口パク、を同じコードで賄う。

## 素材パイプライン

1. 生成: ChatGPT-Image で正面立ち絵を1枚（前髪/輪郭/目鼻口が分かれやすい構図が品質を決める）。
2. 分解: See-through でパーツ別 RGBA ＋隠れ部インペイント＋描画順を自動生成（RTX4060 8GB は NF4 量子化版かクラウドデモ）。
3. 整形: パーツを集約（前髪/後ろ髪/顔/目/口/体程度）、輪郭の奥側を数 px 塗り足し（crack 対策）、Z 深度と描画順を layers.json 化。
4. リグ: 口/目のキーフォームを StretchyStudio で作成しデータ抽出。`tools/` にバッチ化（既存 `slice_character_sheets.py` と同列）。
5. 1体目は手作業分解で基準を作り、自動化は2体目以降に回す。

## 正直な限界（最初から織り込む）

- **±30°超の深い振り向き・横顔は破綻**（板の平面性がバレる。Live2D のメッシュ面変形が無い本質的限界）。緩い首振り（±20〜30°）に狙いを絞り、超過分は離散テクスチャ差分にクロスフェードする保険を入れる。
- **レイヤー境界の crack / halo / 二重輪郭**は不可避の宿題。奥側 edge bleed・回転角制限・Z 段差最小化・α feather・`renderingGroupId` での描画順固定を最初から織り込む。
- **ChatGPT-Image 生成絵の分解品質は未検証**（複雑な髪・アクセは崩れやすい）。1体目は手作業で基準を作る。
- **3D 空間での verlet 髪揺れ**は前例が少なく PoC 必須。

## 段階的ビルド計画

1. **PoC-1**: Babylon の Z 視差だけ検証。手書き 3-4 枚の板を実 Z 配置し、透視カメラ＋faceRoot 回転で振り向きが視差で出るか・破綻角を実測。`01-babylon` が苗床。
2. **PoC-2**: パーツ内変形（口=Morph Targets、目=縦スケール、髪=verlet）を1体の手作業分解素材で確認。
3. **PoC-3**: RigModel を描画非依存の純データ層に切り出し。
4. **PoC-4**: SignalBridge で既存 face signal / WS 中継に結線。
5. **PoC-5**: 素材パイプライン（ChatGPT 絵→See-through→layers.json→Babylon 表示）を通し、分解品質を実データで判定。
6. **PoC-6**: `?obs=1` 透過＋`RigModel` 公開 API を整え、OBS と tommieChat 両方で同一資産を表示。
7. **PoC-7**: character-config に `type:'sheet'｜'rig'` 併存、rig 無いキャラは従来スプライトにフォールバック。
