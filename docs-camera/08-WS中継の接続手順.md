# WS 中継の接続手順（iPhone → サーバ → OBS の CEF）

`camera.html?tx`（送信）と `camera.html?rx`（受信）を中継サーバ（`npm run relay`）でつなぐ手順。
設計の背景は [05-OBSにカメラを触らせない代替案.md](05-OBSにカメラを触らせない代替案.md) を参照。

役割:

- **tx（producer）**: カメラ＋推論を動かし、状態フレームを送る。設定 UI もここ。
- **rx（consumer）**: カメラを起動せず、受信した動きで描画する。OBS のブラウザソース用。
- **中継サーバ**: 受け取った state / config を素通しするだけ（信号の計算はしない）。

すべて `guruguru-avatar/` で実行する。初回は `npm install` を済ませておく。

## 手順A: PC 1台・2タブ（最短・TLS 不要）

`http://localhost` はブラウザが secure context 扱いなので、**localhost ならカメラが TLS なしで動く**。
依存ゼロで「推論 → 送信 → 中継 → 受信 → 描画」を一気に確認できる。

```bash
# ターミナル1: 中継サーバ（平文 ws でよい）
npm run relay            # ws://0.0.0.0:8787

# ターミナル2: dev サーバ
npm run dev
```

ブラウザで2つ開く（必ず `localhost` で。WSL でも Windows のブラウザから `localhost:5173` で届く）。

- 送信側(tx): `http://localhost:5173/camera.html?tx` … カメラを許可して顔を動かす
- 受信側(rx): `http://localhost:5173/camera.html?rx`

確認ポイント:

- tx タブの画面下に「**CEF 接続中（1）**」が出る（rx を閉じると「CEF 未接続」）。
- tx で顔を振る・口を開ける・首をかしげる → rx 側のアバターが同じ動きをする。
- rx はカメラを起動していない（受信した動きだけで描画）。

## 手順B: iPhone → サーバ → OBS の CEF（LAN・TLS あり）

iPhone は別端末なのでカメラに **HTTPS が必須**、WS も **WSS** が要る。
証明書は **Tailscale 証明書を推奨**（iPhone と PC が同じ実証明書を自動で信頼でき、mixed-content も避けられる）。
mkcert を使う場合は両端末にルート CA を入れる。

```bash
# 中継を WSS で起動（同じ鍵を dev にも渡す）
RELAY_CERT=cert.pem RELAY_KEY=key.pem npm run relay

# dev を HTTPS で起動
VITE_TLS_CERT=cert.pem VITE_TLS_KEY=key.pem npm run dev
```

- iPhone（送信）: `https://<PCのLAN名/IP>:5173/camera.html?tx`
- OBS の CEF（受信）: ブラウザソースに `https://<同ホスト>:5173/camera.html?rx&obs=1`
  - `?obs=1` で背景透過のオーバーレイになる。
  - 中継が別ホストなら `&relay=wss://<host>:8787` を付ける（既定はページと同ホストの :8787）。

iPhone 側で「CEF 接続中（n）」が出れば結線 OK。設定（感度・口・ズーム等）は iPhone で変更すると、
変更時に CEF へ送られて反映される（数秒ごとの再送はしない）。

## URL パラメータ早見表

| パラメータ | 意味 |
| --- | --- |
| `?tx` / `?tx=ws` | 送信側（カメラ＋推論＋設定 UI） |
| `?rx` / `?rx=ws` | 受信側（カメラなし・受信描画） |
| `?relay=<url>` | 中継 URL を明示（既定: ページと同ホストの `:8787`） |
| `?obs=1` | 背景透過＋UI 非表示（OBS オーバーレイ用） |

## つまずいたら

- **カメラが出ない**: `localhost` で開いているか。実機なら HTTPS か。WSL の IP 直打ちは
  secure context 外でカメラ不可。
- **CEF が「未接続」のまま**: 中継 URL/ポートと、`ws`↔`wss` の一致を確認。
  HTTPS ページから `ws://` は mixed-content で不可（`wss://` にする）。
- **rx が動かない**: tx 側で「CEF 接続中」になっているか、ブラウザのコンソールに WS エラーが
  出ていないかを確認。証明書未信頼だと WSS 接続が張れない。
- **ポート 8787 が使用中**: `RELAY_PORT=9000 npm run relay` で変更し、rx には
  `&relay=ws(s)://<host>:9000` を付ける。
