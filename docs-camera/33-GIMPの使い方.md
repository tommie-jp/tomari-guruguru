# GIMP の使い方（透過編集メモ）

カメラ版のアバター素材を手直しするときの GIMP 操作メモ。背景や不要部分の透過、
透明部分への描き直し、よくあるハマりどころをまとめる。GIMP 2.10 / 3.0 系が対象。

関連: [31-アバターの追加.md](31-アバターの追加.md) / [32-アバター画像の生成AI.md](32-アバター画像の生成AI.md)

## 事前準備（必須）

透過編集の前に、レイヤーに**アルファチャンネル**を持たせる。これが無いと、消した
場所が透明ではなく背景色（白など）で塗られてしまう。

- メニュー **レイヤー(L) → 透明部分 → アルファチャンネルの追加**
- すでに有効ならグレーアウトしている（＝そのままで OK）。

## 1. 閉じた領域を透過で塗りつぶす

線で囲まれた内側や、同じ色のかたまりをまとめて透明にする。

1. **ファジー選択（マジックワンド）**（`U`）を選ぶ。自由な形なら**自由選択（なげなわ）**（`F`）。
2. 透明にしたい領域の内側をクリックして選択する（点線で表示）。
3. 縁が残るなら **選択(S) → 拡大** で 1〜2px 広げる（消えすぎなら **縮小**）。
4. **編集(E) → 消去**（`Delete`）で透明（市松模様）になる。
5. **選択(S) → なし**（`Ctrl+Shift+A`）で確定。

塗りつぶしツール（バケツ）は色を塗る道具なので透明化には使えない。**選択 → 消去**が基本。

## 2. ブラシでなぞって透過する

### 消しゴム方式（手軽）

1. **消しゴム**（`Shift+E`）を選ぶ。
2. ブラシ・サイズ・不透明度を調整（縁をぼかすなら Hardness を下げる）。
3. 消したい場所をなぞる → アルファ有効なら透明になる。消しすぎは `Ctrl+Z`。

### レイヤーマスク方式（非破壊・おすすめ）

元画像を壊さず、何度でも調整できる。

1. レイヤー右クリック → **レイヤーマスクの追加** → **完全不透明（白）**。
2. 描画色を**黒**にする（`D` で初期化、`X` で前景/背景を入れ替え）。
3. マスク上をブラシ（`P`）で塗る。
   - **黒で塗る = 透明になる** / **白で塗る = 元に戻る** / グレー = 半透明。
4. 確定するなら レイヤー右クリック → **レイヤーマスクの適用**。

## 3. スポイトで拾った色をペンで描く

透明にした場所などへ、画像内の色を拾って描き足す。

1. **色を拾う**: ブラシ描画中に **`Ctrl` を押しながらクリック**で一時的にスポイトになる
   （拾った色は前景色に入る）。専用の**スポイト**ツール（`O`）でも可。
2. **描く**: **鉛筆（`N`）**＝くっきり／**ブラシ（`P`）**＝なめらか、を選んでなぞる。

レイヤーマスクで透過した場合は、マスクではなく**レイヤー本体**を選んでから描くこと
（マスクが選択中だと色は乗らない）。レイヤーパネルで白枠が付いている方がアクティブ。

## よくあるハマりどころ

### 鉛筆・ブラシで描けない

1. **選択範囲が残っている（最多）**: ファジー選択の点線が生きていると、選択の**内側
   だけ**しか描けない。外を描こうとしても無反応。→ **`Ctrl+Shift+A`** で選択解除。
2. **動的特性が「Pressure Size」**: ペンタブの筆圧前提のため、マウスだと線が極細＝
   ほぼ見えない。→ ツールオプションの **動的特性を「なし（Dynamics Off）」** にする。
3. **強さ(Force)が低い**: 薄いときは 100 に上げる。
4. **対象レイヤー違い**: レイヤーパネルで描きたいレイヤー（例 `F.webp`）を選択。

### 描いても透明部分に色が乗らない

| 症状 | 原因 | 対処 |
| --- | --- | --- |
| 消すと白くなる | アルファチャンネル無し | レイヤー → 透明部分 → アルファチャンネルの追加 |
| 透明部分に描けない | 透明保護がオン | レイヤーパネルの鍵（透明部分の保護）をオフ |
| 一部しか効かない | 選択が残っている | 選択 → なし（`Ctrl+Shift+A`） |

## 書き出し（透過を保持）

- **ファイル(F) → 名前を付けてエクスポート**
- 拡張子を **`.png`** または **`.webp`** にする（`.jpg` は透過を保持できない）。
- デフォルト(01-tomari)の個別スライスのみ `public/slices2/`（`<状態>/r<行>c<列>.webp`）。
  02 以降のアバターはシート方式 `public/slices2-sheets/<id>/{A..F}.webp`。
  個別スライスを直接直すならこの形式に合わせる。

## このプロジェクトとの関係

スライス済み画像を1枚ずつ直すより、**元の角度シート（`assets/<id>/{A..F}.png`）を
GIMP で直してから再変換**するほうが整合が取りやすい。

tomari 以外はシート全体（編集対象は `assets/<id>/` などの元シート）を編集 → `doAvatarConvert.sh` で再変換が基本。

1. `assets/<id>/` の A〜F シートを GIMP で編集（透過・描き足し）。
2. `.png` でエクスポートして上書き。
3. `./doAvatarConvert.sh assets/<id> <id>`（既定 fill=0.95）で `public/slices2-sheets/<id>/` へ再変換。

詳細は [31-アバターの追加.md](31-アバターの追加.md) を参照。
