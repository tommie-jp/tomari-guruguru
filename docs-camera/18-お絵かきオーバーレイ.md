# お絵かきオーバーレイ

配信映像（透過オーバーレイ）の上に、フリーハンド・図形の選択／移動・テキスト・**ピクセル単位の
消しゴム**でお絵かきを重ねる機能。`fabric.js` を使う。手元で描いた内容を WS 中継で OBS 側へ送って
表示する（顔ポーズと同じ `/__relay` 経路に相乗り）。

## 仕組み（操作と表示を分ける）

OBS のブラウザソースは基本マウス操作できないので、**操作（描く）と表示（OBS）を別ページに分ける**。

- **操作側（tx / local）**: ふだんのブラウザのタブで `?draw` を付けて開く。ツールバーで描く。
- **表示側（rx = OBS のブラウザソース）**: 受信した内容を再描画するだけ（操作不可・透過）。

```text
[手元 tx]  index.html?tx&draw      ← ツールバーで描く（fabric.Canvas）
   │ 描画が確定するたび {scene, w, h} を送信
   ▼
WS 中継 /__relay （pose と同じ経路。中継は素通し）
   │ 後から OBS が繋がっても現在のシーンを再送（取りこぼし防止）
   ▼
[OBS rx]  index.html?rx&obs        ← 受信して再描画（fabric.StaticCanvas・透過）
```

操作 PC と OBS が同じ Windows 機なら、両方とも `http://localhost:5173/...` で繋げば中継 URL は
共有される（別配線不要）。別 PC 配信は `?relay=<ws(s) URL>` で中継先を明示する
（[16-URLパラメータ一覧.md](16-URLパラメータ一覧.md) の `?relay` 参照）。

## 使い方

お絵かきは**既定で有効**。操作側（`index.html` を普通に開く＝local、または `?tx`）では `?draw` を
付けなくてもツールバーが出る。画面左下のボタン列（カメラ／デバッグ／表情…）の**「お絵かき」ボタン**で
ツールバーの表示と描画入力を ON/OFF できる（描いた線は OFF にしても配信に残る）。完全に無効化したい
ときだけ `?draw=0`。

```text
# 手元（描く側）。普通に開けばお絵かき有効（?tx は OBS へ送る場合）
http://localhost:5173/index.html
http://localhost:5173/index.html?tx

# OBS のブラウザソース（表示側）。受信した絵を常に表示
http://localhost:5173/index.html?rx&obs

# お絵かきを完全に無効化したいとき
http://localhost:5173/index.html?draw=0
```

ツールバー（手元の画面上部中央。「お絵かき」ボタンが ON のときだけ出る）。左端の
**ドラッグハンドル（⠿）をつかんで好きな位置へ移動**できる（位置は localStorage に保存。
ハンドルをダブルクリックで既定位置に戻る）:

- **手** … 描画 OFF。下のアバターをドラッグ／ズームできる（既定）。
- **ペン** … フリーハンド。色・太さスライダーが効く。
- **消し** … ピクセル単位の消しゴム（線の一部だけ消せる。太さ ≒ ペン太さ ×3）。
- **選択** … 描いた物を選んで移動・拡縮・回転。
- **文字** … 空き領域をクリックしてテキストを置き、その場で入力（空のまま抜けると消える）。
- **色 / 太さ** … ペン・文字の色と太さ（localStorage に保存）。
- **戻す** … 直前の確定操作を取り消す。
- **全消し** … すべて消す。

道具を選ぶと描画キャンバスがマウス入力を受ける（その間はアバターを掴めない）。**手**に戻すと
入力が下のアバターへ通る。

## 透過について

- お絵かきキャンバスは背景を塗らない（fabric 既定で透明）。描いていない所はアルファ 0 のまま
  なので、既存のステージモード（`?obs` / `?rx` の透過）にそのまま乗る。
- OBS のブラウザソース設定の「カスタム CSS」既定値
  `body { background-color: rgba(0,0,0,0); }` は**消さない**（消すと不透明になる）。
- 半透明や影もそのまま合成される。背景代わりの不透明な四角を敷かないこと。

## 解像度の違い（手元と OBS でサイズが違うとき）

送信メッセージに手元キャンバスの論理サイズ `w,h` を載せ、OBS 側は `setViewportTransform` で
自分のサイズへ全体を拡縮して合わせる。手元と OBS の縦横比が違うと多少伸縮するが、配信は基本 16:9
なので実用上の問題は小さい。きっちり合わせたいときは手元のウィンドウを OBS のブラウザソースと
同じ解像度（例: 1920×1080）にする。

## セキュリティ

- 中継は無認証 WS（dev 同居は loopback 限定。LAN 公開は `RELAY_EXPOSE=1` のみ）。お絵かきの
  ストロークも盗聴・偽注入は可能（数値 pose と同等のリスク）。信頼できる私設網（tailscale 等）で使う。
- 受信側は描画前にオブジェクト数・サイズを検証してから流す（過大なシーンは無視）。

## 技術メモ

- ライブラリ: `fabric`（v7）＋ `@erase2d/fabric`（消しゴム）。どちらも MIT。
- 消しゴムは消去結果をベクタの `clipPath`（`ClippingGroup`, `type:'clipping'`）として各オブジェクトに
  保存するので、`toObject()` → `loadFromJSON()` で OBS 側へそのまま同期できる。
  `import '@erase2d/fabric'` が `clipping` を classRegistry へ登録する（**操作側・表示側の両方で必須**）。
- 実装: `src/draw-layer.jsx`（描画レイヤー）・`src/draw-mode.js`（`?draw` 解析）。同期は
  `src/face/relay-client.js` / `use-relay.js` の `draw-scene` メッセージ。後着 OBS への再送は
  `need-config`（config と同じ仕組み）に相乗りしている。
- 対象は camera 版（`index.html`）のみ（演出と同様、talk / ぐるぐる版には載せない）。
