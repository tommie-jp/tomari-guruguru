# バージョン管理と表示のしくみ

camera.html 右下に出る `v1.0.0 · f7efa25 · 2026-06-17` の中身と、
バージョンの上げ方をまとめたメモ（2026-06-17 導入）。

関連: [01-使い方.md](01-使い方.md) / [04-OBSでライブ配信.md](04-OBSでライブ配信.md)

## 結論（先に運用だけ知りたい人向け）

- 節目で上げるのは **semver**（`package.json` の `version`）。コマンドは `npm version <patch|minor|major>`。
- タグ間の手動デプロイは、画面の **SHA・日付** が「今ライブに出ているビルド」を正確に示す。
- だから `version` だけを頻繁に上げる必要はない。**節目は semver、間は SHA** の二段構え。

## 表示の3要素

画面右下に `v{semver} · {sha} · {date}` の形で出る（`?obs=1` のステージモードでは非表示）。

| 要素 | 例 | 出どころ | 役割 |
| --- | --- | --- | --- |
| semver | `v1.0.0` | `package.json` の `version` | 人間向けの節目。手動で上げる |
| git short SHA | `f7efa25` | ビルド時の `git rev-parse --short HEAD` | **どのコミットがライブか**を一意に特定 |
| build date | `2026-06-17` | ビルド時刻（UTC 日付） | いつのビルドか |

dev 起動（`npm run dev`）では `v1.0.0 · dev` と表示され、本番ビルドと区別できる。

## しくみ（ビルド時に焼き込む）

[vite.fork.js](../vite.fork.js) の `define` で、ビルド時に定数を静的置換している。

- `__APP_VERSION__` … `package.json` の `version`
- `__GIT_SHA__` … `git rev-parse --short HEAD`。取れなければ CI の環境変数 `GITHUB_SHA`
  の先頭7桁 → それも無ければ `unknown`
- `__BUILD_DATE__` … `new Date()` の `YYYY-MM-DD`

`command === 'build'` のときだけ SHA / 日付を埋め、`serve`（dev）では `dev` にする。

[src/camera-app.jsx](../src/camera-app.jsx) 側はこれらを読んで `VERSION_LABEL` を組み立て、
右下に控えめに表示する。`define` が効かない環境でも落ちないよう `typeof` でガードしている。

```text
build: v1.0.0 · f7efa25 · 2026-06-17
dev:   v1.0.0 · dev
```

### CI（GitHub Actions）での注意

- `pages.yml` は `actions/checkout` 済みなので、CI でも `git rev-parse` で SHA を取得できる。
- 表示される SHA は **ビルドされたコミット**（= `workflow_dispatch` で指定した `main` の HEAD）。
  つまり「今デプロイされているのはどのコミットか」がそのまま画面に出る。

## バージョンの上げ方

semver の指針:

- `patch` … バグ修正・素材差し替えなど後方互換の小修正（`1.0.0 → 1.0.1`）
- `minor` … 後方互換の新機能追加（`1.0.0 → 1.1.0`）
- `major` … 破壊的変更（`1.0.0 → 2.0.0`）

手順:

```bash
# package.json の version を上げ、コミットと git タグ v1.0.1 を自動作成
npm version patch    # or minor / major

# タグも一緒に push する（npm version はローカルにタグを作るだけ）
git push origin main --follow-tags

# 本番反映（手動デプロイ。詳細は doDeploy.sh / デプロイ手順）
./doDeploy.sh
```

注意点:

- `npm version` は作業ツリーがクリーン（未コミット変更なし）でないと失敗する。先にコミットしておく。
- タグは `git push --tags` か `--follow-tags` を付けないと GitHub に上がらない。
- semver を上げなくてもデプロイは可能。その場合 `version` は据え置きで SHA・日付だけが変わる。

## 将来の選択肢（今は不要）

コミットは既に `feat:` / `fix:` / `docs:` の Conventional Commits 風なので、
規模が大きくなったら release-please や changesets で **自動採番・CHANGELOG 生成**に移行できる。
現状は手動 `npm version` で十分。
